# 测试指南

## 集成测试脚本

主测试脚本：

```bash
bash scripts/test_tunnel.sh
```

脚本默认端口：

- 单跳隧道：`19999 -> 19998`
- 双跳 CFB：`19995 -> 19996 -> 19997`

## 覆盖范围

- 单跳隧道数据完整性
  - 短消息
  - 空消息
  - 10KB / 64KB / 1MB 负载
- 单跳隧道并发能力
  - 4 并发客户端
  - 16 并发客户端
- 单跳隧道稳定性
  - 快速连断
- 双跳 CFB 数据完整性
  - 短/空/大负载
  - 16 并发客户端
- 客户端流水线行为
  - 不等待逐包响应，连续发送多段数据

## 退出码

- `0`：全部通过
- 非 `0`：至少一项失败

## 常见问题

- 端口绑定失败时，先确认旧进程已退出。
- 机器负载高时若偶发失败，建议重试并结合隧道日志排查。
- 手工验证可将隧道分终端启动，再用 `nc` 或自定义客户端联调。
